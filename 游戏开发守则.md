# 大饼游乐场 · 游戏开发守则

> 本文件应放置在每个游戏仓库的根目录中，作为开发和评审的核心参照。
>
> 版本：v1.0 ｜ 最后更新：2026-02-17

---

## 我们为什么做这些游戏

为 2~3 岁的大饼创造快乐。不是为了教学指标，不是为了用户增长，就是让一个小朋友在 iPad 上用手指戳一戳，能咯咯笑出来。

一切设计决策的终极判断标准：**大饼玩的时候开不开心？**

---

## 设计原则

### 1. 一个核心交互

每个游戏只允许一种核心操作方式：点击、滑动或拖拽，三选一，不混合。2-3 岁幼儿一次只能掌握一种交互模式。

### 2. 因果即时

手指碰到屏幕的瞬间就必须有反馈。没有加载等待，没有延迟动画。触摸→反馈的延迟不超过 100ms。这是建立"我能影响世界"认知的基础。

### 3. 零惩罚

没有"错误"，没有"失败"，没有"Game Over"，没有倒计时压力。只有"做对了会更开心"和"随便玩也很开心"的区别。不设负面音效、不扣分、不显示红色错误标记。

### 4. 感官丰富

每次有效交互同时触发：视觉变化（动画/颜色）+ 声音反馈（音效/语音）。声音比画面更重要——对 2-3 岁幼儿来说，一个有趣的"啵"声比一段精美的动画更有吸引力。

### 5. 大即正义

所有可交互元素最小 200px，越大越好。元素之间保持充足间距，防止误触。幼儿的手指控制不精确，设计要包容粗放的操作。

### 6. 熟悉感

尽量融入大饼认识的事物：他自己的照片、喜欢的动物、家人的声音、日常生活场景。熟悉感带来安全感，安全感带来探索欲。

### 7. 无限重复

游戏没有强制终点。大饼想玩多久玩多久，重复多少次都行。重复是幼儿学习和掌握技能的核心方式，不是"无聊"的表现。

### 8. 两步可玩

从打开游戏到开始玩，不超过 2 次点击。不要有复杂的菜单、设置页面或加载流程。开屏即玩是最理想的状态。

## 技术指标

### 架构

| 项目 | 要求 |
|------|------|
| 文件结构 | 单文件 HTML（内联 CSS + JS） |
| 外部依赖 | 零依赖，不使用任何框架或 CDN |
| 部署方式 | GitHub Pages，仓库名即游戏标识 |
| 浏览器兼容 | iPad Safari（最优先）、Chrome、移动端浏览器 |

### 交互

| 项目 | 要求 |
|------|------|
| 触屏事件 | 必须使用 `touchstart`，不能只依赖 `click` |
| 点击区域 | 可交互元素 ≥ 200px × 200px |
| 元素间距 | 可交互元素间距 ≥ 64px |
| 响应延迟 | 触摸到视觉/音效反馈 ≤ 100ms |
| 手势类型 | 仅支持单指操作（点击/滑动/拖拽），禁止双指手势 |
| 误触防护 | 设置/退出等非游戏按钮需长按或放在角落小区域 |

### 布局与适配

| 项目 | 要求 |
|------|------|
| 单位 | 使用 `vw` / `vh` / `vmin` 等相对单位，禁止固定 `px` 布局 |
| 适配目标 | iPad 竖屏（768×1024）为主，兼容横屏和手机 |
| viewport | 设置 `<meta name="viewport" content="width=device-width, initial-scale=1.0, user-scalable=no">` |
| 方向处理 | 支持竖屏和横屏，或在不适合的方向给出温和的旋转提示 |

### 视觉

| 项目 | 要求 |
|------|------|
| 色彩 | 鲜明、高对比度、高饱和度 |
| 背景 | 简洁不杂乱，不干扰前景交互元素 |
| 动画帧率 | ≥ 30fps，优先使用 CSS `transform` / `opacity`（GPU 加速） |
| 动画风格 | 流畅自然，避免频闪或高速闪烁（癫痫安全） |
| 图片格式 | PNG（透明背景）或 SVG，头像统一 200×200px |

### 音效

| 项目 | 要求 |
|------|------|
| 实现方式 | Web Audio API（零外部文件）或内联 base64 音频 |
| 正面音效 | 每次有效交互必须有音效反馈 |
| 负面音效 | 禁止使用错误提示音、警告音、失败音效 |
| 语音 | 鼓励使用中文语音（"好厉害""哇""嘻嘻"） |
| 背景音乐 | 轻快风格，必须提供关闭按钮 |
| 音量 | 默认中等音量，不要突然的大声音效 |

### 数据存储

| 项目 | 要求 |
|------|------|
| 存储方式 | localStorage |
| 键名前缀 | `dabing_[游戏名]_` |
| 存储内容 | 最高分、游戏偏好设置 |
| 隐私 | 不收集任何个人数据，不调用任何外部 API |

### 性能

| 项目 | 要求 |
|------|------|
| 首次加载 | ≤ 2 秒（本地/GitHub Pages） |
| 图片 | 预加载所有游戏素材，避免运行时卡顿 |
| 内存 | 避免内存泄漏，及时清理不用的定时器和事件监听 |
| 离线 | 单文件架构天然支持离线，未来可添加 Service Worker |

## 难度分级

每个游戏至少提供"小宝宝"模式，根据游戏类型酌情添加更高难度：

| 等级 | 目标年龄 | 交互元素数量 | 目标停留/响应时间 | 同时出现数 |
|------|---------|------------|----------------|----------|
| 小宝宝 | 2-3 岁 | 2-4 个 | ≥ 4 秒 | 1 个 |
| 幼儿 | 4-5 岁 | 6-9 个 | ≥ 2 秒 | 1-2 个 |
| 挑战 | 6 岁+ | 9 个 | 递减 | 多个 |

大饼当前适用"小宝宝"模式，这是默认模式，必须是进入游戏后的第一选项。

## 上线前自查清单

每个游戏发布前，对照以下清单逐项确认：

- [ ] iPad Safari 上实际触屏测试通过
- [ ] 所有可交互元素 ≥ 200px
- [ ] 使用 `touchstart` 事件
- [ ] 点击后 100ms 内有视觉+音效反馈
- [ ] 无任何负面反馈（错误音效、Game Over、扣分）
- [ ] 2 次点击内可开始游玩
- [ ] 横竖屏均可正常显示
- [ ] 背景音乐有关闭按钮
- [ ] localStorage 键名使用 `dabing_` 前缀
- [ ] 无外部依赖、无外部网络请求
- [ ] 单文件 HTML 可独立运行
- [ ] 让大饼实际试玩并观察反应
